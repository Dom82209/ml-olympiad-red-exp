# Объяснятель: выравнивание языковой модели

Проект по треку «Искусственный интеллект»: дообучение компактной языковой модели, чтобы она по умолчанию объясняла сложные темы простым языком.

В полном пайплайне предусмотрены пять этапов:

1. SFT для переноса простого стиля.
2. DPO по стилю для закрепления предпочтения простого ответа.
3. Reward Model для оценки качества объяснений.
4. DPO по качеству.
5. SimPO по качеству.

Ноутбук будет дополняться по ходу выполнения задач отдельными коммитами.

## Задача 1 — SFT

Основной файл:

- `task1_sft_colab.ipynb`

На первом этапе модель `Qwen/Qwen3-4B-Instruct-2507` дообучается через QLoRA 4-bit на датасете `kid_adult.jsonl`.

Что делает ноутбук для задачи 1:

- запускается сверху вниз в Colab/Kaggle с GPU;
- фиксирует `seed = 42`;
- обучает QLoRA 4-bit адаптер `Qwen/Qwen3-4B-Instruct-2507` на `kid_adult.jsonl`;
- генерирует ответы на `public_test_style.jsonl` с `do_sample=False`;
- печатает `Task 1 SFT P_simple = ...`;
- печатает букву интервала.

Метрика задачи 1:

- средний `P_simple` ответов на `public_test_style.jsonl`;
- генерация greedy: `do_sample=False`;
- ответ выбирается по интервалу, в который попало значение `P_simple`.

Данные и измеритель для первой задачи лежат в `task_data/`.

## Задача 2 — DPO по стилю

На втором этапе поверх SFT-модели запускается DPO на том же датасете `kid_adult.jsonl`.

Настройка пар:

- `chosen = kid`;
- `rejected = adult`.

Метрика задачи 2:

- средний `P_simple` ответов на `public_test_style.jsonl`;
- генерация greedy: `do_sample=False`;
- ноутбук печатает `Task 2 Style DPO P_simple = ...`;
- ответ выбирается по интервалу, в который попало значение `P_simple`.

## Задача 3 — Reward Model

На третьем этапе обучается отдельная модель-оценщик качества на датасете `good_bad.jsonl`.

Настройка пар:

- `chosen` — хороший простой ответ;
- `rejected` — плохой простой ответ.

Метрика задачи 3:

- pairwise accuracy на `public_test_quality.jsonl`;
- считается доля пар, где reward для `chosen` выше reward для `rejected`;
- ноутбук печатает `Task 3 Reward Model pairwise accuracy = ...`;
- ответ выбирается по интервалу, в который попало значение accuracy.

## Задача 4 — DPO по качеству

На четвёртом этапе style-aligned модель дополнительно оптимизируется на датасете `good_bad.jsonl`.

Настройка пар:

- `chosen` — хороший ответ;
- `rejected` — плохой ответ.

Метрика задачи 4:

- implicit-preference accuracy на `public_test_quality.jsonl`;
- генерация не используется;
- для каждого ответа считается средний logprob на токен ответа;
- ноутбук печатает `Task 4 Quality DPO implicit-preference accuracy = ...`;
- ответ выбирается по интервалу, в который попало значение accuracy.


## Задача 5 — SimPO по качеству

На пятом этапе выполняется SimPO на том же датасете `good_bad.jsonl`.

Настройка пар:

- `chosen` — хороший ответ;
- `rejected` — плохой ответ.

Отличие от задачи 4:

- отдельная reference-модель не используется;
- loss строится на разнице средних logprob текущей модели;
- это снижает потребление памяти по сравнению с DPO.

Метрика задачи 5:

- implicit-preference accuracy на `public_test_quality.jsonl`;
- генерация не используется;
- для каждого ответа считается средний logprob на токен ответа;
- ноутбук печатает `Task 5 SimPO implicit-preference accuracy = ...`;
- дополнительно печатается сравнение с метрикой задачи 4;
- ответ выбирается по интервалу, в который попало значение accuracy.
